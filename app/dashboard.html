💡 Ключевой вывод — одинаковый паттерн вызова
Оба проекта используют HTTP → Router → Singleton → Agent.method(input, context_id) → dict

Разница только в типе pipeline внутри:
• IR-Agent: итеративный ReAct loop — Thought → Action → Observation × N шагов (LLM-driven)
• SQLi Agent: последовательный 7-слойный ensemble — Normalize → Lexical → ML → Semantic → Decision → Severity → Explain

Потенциал интеграции: IR-Agent может вызывать SQLi-агент как инструмент check_sqli(payload) в своём ReAct loop — когда агент анализирует веб-атаку, он может делегировать проверку SQL-инъекций в специализированный pipeline.
